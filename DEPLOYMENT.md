# MoodFood Deployment Guide

## Quick Deploy (Recommended)

### 1. Deploy Backend to Railway (5 minutes)

```bash
cd backend

# Login to Railway (install CLI first: npm i -g @railway/cli)
railway login

# Initialize project
railway init

# Add PostgreSQL database
railway add --database

# Deploy
railway up
```

**Get your backend URL:**
```bash
railway domain
# Output: https://moodfood-api.up.railway.app
```

### 2. Update Frontend API URL

Edit `frontend/vercel.json`:
```json
{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "https://YOUR-RAILWAY-APP.up.railway.app/api/$1"
    }
  ]
}
```

### 3. Deploy Frontend to Vercel (2 minutes)

```bash
cd frontend

# Install Vercel CLI if needed
npm i -g vercel

# Deploy
vercel

# Follow prompts, set framework to Vite
```

**Your app is live!** 🎉

## Detailed Setup

### Backend Environment Variables (Railway)

In Railway dashboard, add these variables:

```
NODE_ENV=production
DATABASE_URL=${{Postgres.DATABASE_URL}}  # Auto-generated by Railway
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-secure-password
```

### Alternative: Deploy Backend to Render

1. Go to [render.com](https://render.com)
2. Create New Web Service
3. Connect your GitHub repo
4. Settings:
   - **Build Command:** `npm install`
   - **Start Command:** `node server.js`
   - **Environment:** Node
5. Add PostgreSQL database (Free tier)
6. Add environment variables

### Alternative: Deploy Backend to Heroku

```bash
# Install Heroku CLI and login
heroku login

# Create app
heroku create moodfood-api

# Add PostgreSQL
heroku addons:create heroku-postgresql:mini

# Deploy
git subtree push --prefix backend heroku main

# Set environment variables
heroku config:set NODE_ENV=production
heroku config:set ADMIN_PASSWORD=your-password
```

## Custom Domain Setup

### Vercel (Frontend)
1. Go to Project Settings → Domains
2. Add your domain
3. Follow DNS instructions

### Railway (Backend)
1. Go to Settings → Domains
2. Add custom domain
3. Configure DNS CNAME record

## Troubleshooting

### Backend won't start
```bash
# Check logs
railway logs

# Verify database connection
curl https://your-api.com/api/health
```

### Frontend can't reach backend
1. Verify API URL in `vercel.json`
2. Check CORS is enabled in backend
3. Ensure backend is deployed and running

### Database connection errors
1. Verify `DATABASE_URL` is set
2. Check Railway PostgreSQL is provisioned
3. Try restarting the service
